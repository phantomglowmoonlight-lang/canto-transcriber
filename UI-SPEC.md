## 1. 頁面結構總覽

```
/home                    → 上傳頁面
/home?task=xxx           → 處理中頁面（含進度）
/result/{task_id}        → 結果工作區（主頁面）
/result/{task_id}?export → 匯出對話框（疊層）
/result/{task_id}?report → AI 報告對話框（疊層）
```

> 路由僅供參考，程序員可改用 SPA 內部狀態管理（React Router / Vue Router）。

## 2. 上傳頁面

### 2.1 佈局

```
┌─────────────────────────────────────────────┐
│                                             │
│         🎙 會議錄音轉文字                    │
│                                             │
│     ┌─────────────────────────────┐        │
│     │                             │        │
│     │    📁 拖曳音檔至此           │        │
│     │        或點撃選擇            │        │
│     │                             │        │
│     └─────────────────────────────┘        │
│                                             │
│     支援格式：MP3 / WAV / M4A / OGG / FLAC  │
│                                             │
│     ┌─ 說話人分離 ──────────────┐          │
│     │ ○ VAD 輕量模式（建議）     │          │
│     │ ○ pyannote 精準模式       │          │
│     └───────────────────────────┘          │
│                                             │
│            [ 開始轉寫 ]                      │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.2 行為

| 狀態 | 行為 |
|------|------|
| 未選擇檔案 | 按鈕顯示「開始轉寫」，disabled |
| 檔案拖入 | 拖曳區高亮（藍色邊框 + 淺藍底），放開後顯示檔名與長度 |
| 選擇檔案後 | 顯示「已選擇：xxx.mp3（1 小時 23 分鐘）」+ 按鈕啟用 |
| 不支援格式 | 拖曳區紅色邊框 + toast「不支援的格式：.wmv」 |
| 點擊「開始轉寫」 | 跳轉到 `/home?task={task_id}`（處理中頁面） |

### 2.3 驗證（前端）

- 副檔名白名單：`.mp3`, `.wav`, `.m4a`, `.ogg`, `.flac`, `.aac`
- 檔案大小 > 0 byte
- ffmpeg 可用性（呼叫 `/v1/health`，若不可用在頂部顯示紅色 banner「需要安裝 ffmpeg」+ 下載連結）
- 檔案不要太大，不要太小。這不是在瀏覽器中判定的，而是上傳後由後端判斷。後端可以在 `/v1/transcribe` 上傳回應中觸發，前端接收錯誤後顯示 toast。

## 3. 轉寫處理中頁面

### 3.1 佈局

```
┌─────────────────────────────────────────────┐
│                                             │
│         正在處理會議錄音...                   │
│                                             │
│     ████████████████░░░░░░░░  52%           │
│                                             │
│     階段：語音識別（3 / 4 段完成）            │
│     檔案：會議_2024_10_15.mp3                │
│     長度：1 小時 23 分鐘                      │
│     語言：粵語（yue）                        │
│     預估剩餘：約 12 分鐘                      │
│                                             │
│     ┌─ 處理記錄 ─────────────────┐          │
│     │ ✓ 格式轉換                  │          │
│     │ ✓ 音檔切割（分為 4 段）      │          │
│     │ → 語音識別（第 3 段）       │          │
│     │   說話人分離                 │          │
│     └────────────────────────────┘          │
│                                             │
└─────────────────────────────────────────────┘
```

### 3.2 行為

| 狀態 | 行為 |
|------|------|
| 輪詢中 | 每 2 秒 GET `/v1/tasks/{task_id}`，更新進度 |
| 完成 | 自動跳轉 `/result/{task_id}` |
| 失敗 | 進度條變紅，顯示錯誤訊息 + [重試] 按鈕 |
| 用戶關閉頁面 | 任務繼續在後台運行，再次打開 `/result/{task_id}` 即可查看結果 |

### 3.3 階段對應

| 階段文字 | 對應 progress 範圍 |
|---------|-------------------|
| 格式轉換 + 音檔切割 | 0% – 10% |
| 語音識別 | 10% – 80% |
| 說話人分離 | 80% – 95% |
| 合併與整理 | 95% – 100% |

> 程序員：後端如果無法提供精確階段，至少提供 `progress`（0.0~1.0），由前端自行估算顯示。

## 4. 結果工作區（主頁面）

這是整個系統最核心的頁面，需要最詳細的規範。

### 4.1 整體佈局

```

├────────────────────────────┬──────────────────────────┤
│                            │  發言人                   │
│  檢視模式：[原文|書面語|對照] │  ┌──────────────────┐  │
│                            │  │ 人物 #1 (23次)   │  │
│  ┌───────────────────────┐ │  │ [____陳大明___] │  │
│  │ ☐ 0:00:00  人物 #1    │ │  ├──────────────────┤  │
│  │ 大家好，今日開會         │ │  │ 人物 #2 (18次)   │  │
│  ├───────────────────────┤ │  │ [____李小華___] │  │
│  │ ☐ 0:00:08  人物 #2    │ │  ├──────────────────┤  │
│  │ 我諗我哋要討論一下個預算   │ │  │ 人物 #3 (5次)    │  │
│  │ (書)我想我們要討論預算    │ │  │ [未命名______] │  │
│  ├───────────────────────┤ │  └──────────────────┘  │
│  │ ☐ 0:00:22  人物 #1    │ │                          │
│  │ 冇問題，你繼續講         │ │                          │
│  ├───────────────────────┤ │                          │
│  │ ☐ 0:00:30  人物 #2    │ │                          │
│  │ ...                    │ │                          │
│  └───────────────────────┘ │                          │
│                            │                          │
├────────────────────────────┴──────────────────────────┤
│  [⏮] [▶/⏸] [⏭]  ───●─────────────────  1:23:45     │
│  音檔播放器（底欄）                                     │
└──────────────────────────────────────────────────────┘
```

**佈局比例**：
- 頂部工具列：固定高度 48px
- 主區域橫向二分：段落列表 70% / 發言人面板 30%（最小寬度 260px，最大 400px）
- 底欄播放器：固定高度 64px

### 4.2 頂部工具列

```
┌──────────────────────────────────────────────────────┐
│ ← 返回   會議_2024_10_15.mp3    [翻譯書面語] [AI報告] [匯出 ▾] │
└──────────────────────────────────────────────────────┘
```

| 元件 | 行為 |
|------|------|
| **← 返回** | 回到上傳頁面（確認對話框「未儲存的變更將會遺失，確定離開？」） |
| **檔名顯示** | 不可編輯，灰字顯示原始檔名 + 總時長（如：`會議_2024_10_15.mp3（1:23:45）`） |
| **翻譯書面語** | 點擊後觸發整篇翻譯。翻譯中顯示 spinner；完成後自動切換到「書面語」檢視模式。若已翻譯過，按鈕顯示「重新翻譯」 |
| **AI 報告** | 打開 AI 報告對話框（見第 6 章） |
| **匯出 ▾** | 下拉選單：`匯出 TXT` / `匯出 DOCX`，點擊後打開匯出對話框（見第 5 章） |

**翻譯按鈕的狀態**：

| 狀態 | 按鈕文字 | 樣式 |
|------|---------|------|
| 未翻譯 | 「翻譯為書面語」 | 主色按鈕 |
| 翻譯中 | 「翻譯中 ⏳」 | disabled + spinner |
| 已翻譯（無過期） | 「重新翻譯」 | 次要按鈕 |
| 已翻譯（有過期） | 「重新翻譯 ⚠」 | 次要按鈕 + 橙色邊框，hover 提示「部分句子因重新識別而過期」 |

### 4.3 檢視模式切換

```
[ 原文  |  書面語  |  對照 ]
   ●         ○         ○
```

三選一的 segmented control / button group。與翻譯狀態互動：

| 翻譯狀態 | 可用模式 |
|---------|---------|
| 未翻譯 | 只能選「原文」，「書面語」和「對照」disabled + tooltip「請先翻譯」 |
| 已翻譯 | 三個都可選 |
| 部分過期 | 三個都可選，但過期的句子在「書面語」和「對照」模式中顯示⚠標記 |

### 4.4 段落列表（主區域）

#### 4.4.1 段落行結構

```
┌─ 原文模式 ────────────────────────────────────────────┐
│ ☐  00:00:08  ┌──────────┐                            │
│              │ 人物 #2  │  我諗我哋要討論一下個預算      │
│              └──────────┘                            │
├──────────────────────────────────────────────────────┤
│ ☑  00:00:22  ┌──────────┐                            │
│              │ 人物 #1  │  冇問題，你繼續講              │
│              └──────────┘                            │
│    [🔊降噪 OFF] [📢提音量 OFF]  [🔄重新識別]           │
└──────────────────────────────────────────────────────┘
```

#### 段落行元素說明

| 元素 | 說明 |
|------|------|
| **☐ 勾選框** | 單選（點擊該行任意位置亦可勾選）。Ctrl+點擊可追加多選。Shift+點擊可選取範圍。被選中的行有淺藍底色。 |
| **時間戳** | 格式 `HH:MM:SS`，灰色小字（12px），不可點擊。若有音檔播放，當前播放到的段落自動高亮（淺黃底色）。 |
| **發言人徽章** | 圓角矩形，內含「人物 #N」或已命名的姓名。顏色：每個人物對應一種顏色（共 10 種色盤，自動分配）。已命名的人物徽章右側有鉛筆圖示，點擊可改名。 |
| **文字內容** | 左對齊，14-15px 字級。行距 1.5。原文模式下顯示粵語口語。 |
| **逐句精修工具列** | 只有當段落被選取時才顯示（見 4.6）。 |

```
┌─ 書面語模式 ──────────────────────────────────────────┐
│ ☐  00:00:08  ┌──────────┐                            │
│              │ 人物 #2  │  我想我們要討論一下預算          │
│              └──────────┘                            │
└──────────────────────────────────────────────────────┘
```

```
┌─ 對照模式 ────────────────────────────────────────────┐
│ ☐  00:00:08  ┌──────────┐                            │
│              │ 人物 #2  │  我諗我哋要討論一下個預算      │
│              └──────────┘                            │
│              書面語：我想我們要討論一下預算                │
│                                            ⚠ 過期     │
└──────────────────────────────────────────────────────┘
```

**對照模式說明**：
- 上半行：粵語原文（預設字級）
- 下半行：書面語翻譯（灰色斜體，稍微內縮 8px）
- 如果 `translation_stale: true`，在該行右側顯示橙色 `⚠ 過期` 標籤

#### 4.4.2 段落合併顯示規則

若同一說話人連續發言，且中間間隔 < 2 秒：合併顯示為一個段落方塊，減少視覺碎片化。
但後端 API 應該保留每個獨立 segment，合併在前端做。

合併後：
- 時間戳取第一個 segment 的 start
- 發言人徽章一個
- 文字用空行（或段距）分隔每句

```
┌──────────────────────────────────────────────────────┐
│ ☐  00:01:30  ┌──────────┐                            │
│              │ 人物 #1  │  大家早晨，今日開會主要傾三件事  │
│              └──────────┘                            │
│                       首先係預算嗰邊有啲變動              │
│                       然後係新項目嘅時間表               │
│                       最後係人事安排                     │
└──────────────────────────────────────────────────────┘
```

**合併閾值**：前端可設定參數 `MERGE_GAP_SECONDS = 2`，存於前端的常數

### 4.5 段落選擇與批次操作

#### 選擇規則

| 操作 | 效果 |
|------|------|
| 點擊段落行 | 選取該段落，取消其他選取 |
| Ctrl + 點擊 | 追加 / 取消該段落（toggle） |
| Shift + 點擊 | 選取從上一個選取點到該行的連續範圍 |
| 按 Escape | 取消所有選取 |

#### 選取後的視覺回饋

- 選取的段落行：淺藍底色（`#EBF5FB`）
- 選取的勾選框：☑（填入主色）
- 底部浮現「逐句精修工具列」（見 4.6）
- 狀態欄顯示「已選取 N 句」

### 4.6 逐句精修工具列

選取一個或多個段落後，在選取範圍下方浮現：

```
┌──────────────────────────────────────────────────────┐
│ 已選取 3 句                                           │
│ [ 🔊 降噪 ]  [ 📢 提音量 ]    [ 🔄 重新識別所選段落 ]  │
└──────────────────────────────────────────────────────┘
```

| 元件 | 類型 | 行為 |
|------|------|------|
| **降噪** | Toggle 按鈕 | 點擊在 ON/OFF 之間切換。ON 時背景為藍色、文字白色；OFF 時灰色邊框。狀態為「已選取段落的處理設定」，不是全域的。 |
| **提音量** | Toggle 按鈕 | 同上，獨立於降噪。兩者可同時 ON。 |
| **重新識別所選段落** | 主按鈕 | 點擊後發送 POST `/v1/tasks/{task_id}/retranscribe`，包含所選 segment_ids 和目前的 noise_reduction / volume_boost 布林值。 |

#### 處理進行中

```
┌──────────────────────────────────────────────────────┐
│ ⏳ 正在重新識別所選段落...                              │
│     🟢 第 1 句完成  ●  第 2 句處理中  ○  第 3 句等待  │
└──────────────────────────────────────────────────────┘
```

處理完成後：
- 該工具列自動收起
- 對應段落的文字更新
- 若有書面語翻譯，該段落的 `translation_stale` 變為 true
- Toast 提示「3 句已更新」

#### 復原

每次重新識別前，記錄原始文字。在工具列右側提供小連結「復原」，點擊可還原至上一個版本。最多保留 1 次歷史（每個 segment 各自保留）。

### 4.7 發言人面板（側欄）

```
┌─────────────────┐
│ 發言人          │
├─────────────────┤
│ 🟢 人物 #1      │  ← 顏色圓點 + 代號
│ [ 陳大明______] │  ← 文字輸入框，placeholder 顯示代號
│ 發言 23 次      │
├─────────────────┤
│ 🟡 人物 #2      │
│ [ 李小華______] │
│ 發言 18 次      │
├─────────────────┤
│ 🔴 人物 #3      │
│ [ 點擊命名___ ] │  ← 未命名時的 placeholder
│ 發言 5 次       │
├─────────────────┤
│                 │
│ [ 重置所有名稱 ] │  ← 灰字連結
└─────────────────┘
```

#### 行為

| 操作 | 行為 |
|------|------|
| 在輸入框輸入姓名 + Enter | 更新該發言人的姓名，全篇段落中的該發言人徽章即時更新 |
| 兩個發言人輸入相同姓名 | 輸入框邊框變紅 + 下方顯示警告「人物 #2 和 #3 已被命名為相同姓名，是否合併？」 + [合併] 按鈕 |
| 點擊 [合併] | 將兩個發言人的所有段落合併為同一人 |
| 點擊 [重置所有名稱] | 所有已命名的發言人恢復為「人物 #N」代號（需二次確認） |
| 點擊顏色圓點 | 可循環切換該發言人的顏色（10 色色盤） |

#### 排序

發言人按發言次數由多到少排序。若次數相同，按首次出現時間由早到晚排序。

### 4.8 音檔播放器（底欄）

```
┌──────────────────────────────────────────────────────┐
│ ⏮   ▶ / ⏸   ⏭     ──────●────────────────   1:23:45 │
│         00:04:32                                     │
└──────────────────────────────────────────────────────┘
```

| 元件 | 行為 |
|------|------|
| **⏮ / ⏭** | 跳到上一段 / 下一段的開始時間 |
| **▶ / ⏸** | 播放 / 暫停 |
| **進度條** | 可拖曳定位。拖曳時顯示預覽時間。 |
| **當前時間** | 格式 `HH:MM:SS`，右側顯示總時長 |
| **當前播放段落高亮** | 播放到的段落自動滾動到列表中央，段落行設淺黃底色 + 左側藍色豎線指示器 |

#### 段落同步

播放器與段落列表雙向同步：
- 點擊段落的時間戳 → 播放器跳到該時間開始播放
- 播放器播放到某時間 → 對應段落自動高亮並滾動到視區

## 5. 匯出對話框

### 5.1 觸發

點擊頂部工具列 `[匯出 ▾]` 選單中的 `匯出 TXT` 或 `匯出 DOCX`。

### 5.2 佈局


## 7. 元件狀態規範

### 7.1 通用狀態

| 狀態 | 視覺表現 |
|------|---------|
| **載入中** | spinner 動畫 + 灰色文字描述（如「載入中...」） |
| **空狀態** | 圖示 + 主要文字 + 次要行動指引（如「尚無轉寫結果」[返回上傳]） |
| **錯誤** | 紅色邊框 / 紅色文字，附帶錯誤碼與 [重試] 按鈕 |
| **禁用** | 透明度 40%，cursor: not-allowed，可附 tooltip 說明原因 |
| **已選取** | 淺藍底色（`#EBF5FB`），左側 3px 主色指示線 |

### 7.2 段落行的特殊狀態

| 狀態 | 視覺表現 |
|------|---------|
| **正常** | 白底，細灰分隔線 |
| **已選取** | 淺藍底 + 勾選框填滿 |
| **播放中** | 淺黃底（`#FFF8E1`）+ 左側藍色指示豎線 |
| **翻譯過期** | 橙色 ⚠ 標籤在行末 |
| **重新識別中** | 該行文字閃爍淺綠底色 + 右側 spinner |
| **連續發言（合併顯示）** | 各句之間無分隔線，僅以行距區分 |

### 7.3 Toast 通知

右下角浮現，3 秒後自動消失。三種類型：

| 類型 | 圖示 | 顏色 | 範例 |
|------|------|------|------|
| 成功 | ✅ | 綠色邊框 | 「3 句已更新」 |
| 警告 | ⚠️ | 橙色邊框 | 「部分段落翻譯已過期」 |
| 錯誤 | ❌ | 紅色邊框 | 「重新識別失敗：音檔片段過短」 |

## 10. 前端技術建議（非強制）

以下僅供程序員參考，不構成強制約束，可依實際情況自行決定。

### 10.1 建議架構

| 層級 | 建議 | 理由 |
|------|------|------|
| 框架 | React 18+ 或 Vue 3 | 生態成熟、元件化清晰 |
| 狀態管理 | React Query / TanStack Query（React）或 Pinia（Vue） | 自動處理 API 輪詢、快取 |
| CSS | Tailwind CSS 或 CSS Modules | 快速開發，不引入過重 UI 庫 |
| 音頻播放 | HTML5 `<audio>` + 自訂控制項 | 零依賴，跨瀏覽器 |
| HTTP Client | fetch 或 ky | 輕量，無需 axios 的額外體積 |
| 拖曳上傳 | 原生 Drag & Drop API | 避免 react-dropzone 等依賴 |

### 10.2 API 輪詢策略

```javascript
// 處理中頁面：2 秒輪詢 /v1/tasks/{task_id}
const POLL_INTERVAL = 2000;

// 連續失敗３次 → 停止輪詢，顯示錯誤
const MAX_CONSECUTIVE_FAILURES = 3;
```

### 10.3 打包目標

- 單頁應用（SPA），靜態檔案由 FastAPI 伺服。
- 不需要 SSR。不需要 PWA（本機運行，無離線需求）。
- 若選 React，可用 Vite 打包。
